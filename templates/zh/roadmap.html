路线图

  • 网络启动阶段

    启动项目公测

    • 启动原始网络
    • 首次发布Veles Core R1代码
    • 重构并发布Veles Core R2完整代码

  • 启动成熟网络

    正式启动当前成熟网络的主网

    • 基于比特币Core v0.17重构Veles Core R3的完整代码，并支持多算法和主节点的FXTC
    • 为早期实践者提供50000 VLS的代币兑换奖励
    • 销毁原始预挖部分
    • 直到VCIP01完成阶段的最小区块奖励

  • VCIP01 PoW挖矿阶段

    发布Veles Core v0.17 “Aged Amnesia”

    在启动阶段结束，并进行主PoW共识升级后开始全面挖矿奖励

    • 大量增加区块奖励以提供所需的代币供应量
    • 非线性减半间隔可缩短通胀期
    • 在所有算法中均衡动态奖励
    • 具有减半重新调度和自动动态奖励促进的自适应供给控制机制
    • 主节点奖励在5年内逐步从5％增加到60％
    • 开发团队奖励在5年内从5％逐渐减少到0％

  • Veles主节点安装器

    发布Veles主节点安装程序以简化Linux系统的主节点安装过程

    • Veles主节点安装程序以简化Linux系统的主节点安装过程
    • 支持多操作系统平台，包括Ubuntu，Debian，Gentoo，RedHat，Fedora，CentOS等
    • 也可以使用流行的安装脚本Nodemaster作为Veles fork的替代解决方案

  • 图形界面版本的客户端升级

    全面重新设计适用于所有桌面平台的图形化钱包客户端，支持轻松操作的新功能

    • 为所有桌面平台完全重新设计钱包界面
    • 清晰指示出是使用预发布版本还是稳定版本
    • 支持导出和加载自定义QSS样式表，使设计人员无需C++知识即可轻松定制
    • 开始开发基于Angular的独立实验界面

  • 减少最大供应量

    正式宣布将最大供应量从12,500,000减少到2,500,000 VLS（没有破坏社区共识）

    • 根据VCIP01建立的挖矿共识，将12,500,000VLS的最大供应量减少至2,500,000 VLS，以匹配当前的理论最大供应量
    • 保持完整且具有前瞻性的供应曲线，并确保降低VLS的通货膨胀率
    • 制定计划，在下一个主要版本（v0.18）中减少Veles 主节点抵押代币，从2000降低到500 VLS以匹配新的供应限制，以及正式执行的最大供应量限制

  • Veles Core 升级至 v0.17.1

    Veles Core客户端和服务器的下一个稳定版本，包括所有近期更新和修复的内容

    适用于大多数主流桌面和服务器平台的新安装向导，包括Raspberry PI。

    • Veles Core团队修复了最新测试中出现的问题
    • 从比特币Core v0.17.1移植所有更新内容
    • 支持Travis持续集成和自动化测试
    • Windows 32/64位系统的安装向导
    • 为Ubuntu/Debian/Linux安装.deb软件包和apt-get存储库
    • Raspberry PI vanilla Linux和Raspbian .deb软件包的二进制文件
    • 为macOS操作系统安装 .dmg文件包
    • 修订和更新项目文档，添加了关于主节点的文档

  • Veles Core Web和基础架构

    与Veles 区块链集成的新网站和区块浏览器，用来提供有关网络的实时数据

    新的高负载REST API和高速websocket服务器推送

    • 提供最新信息和全新设计的新项目网站
    • 重新设计的区块浏览器并与网站集成
    • 使用websockets和server-push提供区块链的实时数据和统计数据
    • 提供负载均衡的公共API的高负载后端服务
    • 白皮书和网站的各国语言版本

  • VCIP02 PoS实施阶段

    发布Veles Core v0.18“Blockchain Baracuda”

    实现并启动PoS机制

    • 实现并启动PoS共识机制，作为PoW的补充，从而建立更强大，更安全的网络
    • 激活隐私发送 / 即时发送
    • 将代码重新引入最新的比特币Core v0.18，实现几个新功能作为钱包备份实用程序，并为将来支持硬件钱包奠定基础
    • 新的主节点质押（从2000减少到500 VLS）转换期，以匹配最大的供应量
    • 进一步改善界面客户端的内部支持
    • 在主节点自主选择运行的内测版dVPN的dapp，以支持将来的实时测试

  • VCIP03 dVPN的整合阶段

    更新区块链和网络，为DIP0002特殊事务提供满足dVPN服务需求的最佳环境。用于测试和开发的内测版本的dVPN服务

    • 端口DIP0002特殊事务，以正确的方式开启在区块链上存储VPN订阅
    • 集成DIP0003的确定性主节点，可靠的链上主节点同步
    • 开发用于dVPN连接的RSA证书的分布式授权审核
    • 开始在生态系统内开发DEX（去中心化交易所），以实现代表VPN连接时间的代币交易，从而提供更好的价值存储，价格发现和预付费VPN时间的免费交易
    • 用于测试和客户端开发的志愿者主节点，配置dVPN dapp

  • dVPN实施阶段

    • Tier II 主节点实现用作提供多级路由功能的出口节点
    • IPv6到IPv4路由层以支持IPv6出口节点
    • 区块链订阅管理
    • 桌面和移动版本集成VPN的dVPN客户端钱包
    • 使用志愿者节点做dVPN公开内测版本测试

  • dVPN启动阶段

    过渡期后，所有主节点启用dVPN服务

    官方网络在第一季度正式推出dVPN供公众使用

    Veles生态系统中DEX的公共内测版本

    • dVPN上线供公众使用
    • 主节点运行dVPN dapp必须要有转换周期
    • IPv6到IPv4路由层以支持IPv6出口节点
    • dVPN上线供公众使用
    • 用于购买dVPN连接时间的代币交易DEX公共alpha测试版
    • 探索DEX上的算力租赁代币化和交易的理念，它还可以提供有关算力市场价格的分布式数据，以供更合适的动态PoW奖励调整使用

  • dVPN拓展阶段

    使用来自季度一的数据，开发Proof-Of-Geolocation（地理定位证明），改进dVPN以实现更好的扩展性

    发布Veles DEX公测版

    • 改进和重构dVPN以实现更好的扩展性
    • 使用季度一的数据研究新的扩展可行性解决方案
    • 开始开发和测试地理定位证明的概念，以便在全球范围内实现更加分布式和更优质地分发VLS出口节点
    • dVPN上线供公众使用
    • 公开发布Veles DEX和web界面

  • 引导治理机制阶段

    开发分布式的基础设施，流程和智能合约，以实现未来社区对项目的治理

    • 为Veles生态系统维护和开发所涉及的所有决策过程开发公平且分布式的治理方式和基础设施

  • 社区治理过渡期

    实施并优化治理测试流程

    dVPN网络的主要和改进，新功能，集成和理念

    • 在Veles核心团队的支持下，在实现完全分布式的组织运作之前，建立时间窗口，对所有治理流程进行优化测试
    • 维护dVPN网络并实现社区用户，节点运营商和开发人员建议的新功能，理念，集成或用例

  • 社区治理的彻底移交

    此时，网络上的所有治理和代码的开发应该完全由网络参与者，支持者和开发者社区主持的分布式组织掌握

    • Veles核心团队的现任成员将继续帮助社区开展Veles生态系统开发，因为我们热衷于尽自己最大的能力不断改变世界
